## 1. 系统简介

### 1.1 产品概述
无人船ROS2多机实时通讯与目标自动检测系统是一套基于ROS2框架的无人水面艇仿真与感知平台。系统通过研华工控机（Ubuntu）和仿真计算机（Windows）的协同工作，实现高性能的实时数据通信、3D仿真和目标自动检测。

### 1.2 主要功能
- ✅ **双机实时通信**: Ubuntu和Windows之间的低延迟ROS2数据传输
- ✅ **图像数据传输**: 支持30Hz的高频RGB图像传输
- ✅ **3D仿真环境**: 基于UE5的高保真海洋场景仿真
- ✅ **动力学仿真**: 基于Simulink的船舶运动仿真
- ✅ **目标检测**: 3D包围盒投影与目标识别

### 1.3 系统架构

```
[Ubuntu工控机]  ←──── 网线直连 ────→  [Windows仿真机]
    • 感知计算                           • UE5 3D仿真
    • 目标检测                           • Simulink动力学仿真
    • 控制决策                           • 图像生成
    192.168.100.1                       192.168.100.2
```

### 1.4 适用用户
- 机器人研究人员
- 无人船系统开发者
- 高校科研团队
- 仿真测试工程师

## 2. 安装指南

### 2.1 硬件要求

#### Ubuntu工控机（推荐配置）
- **CPU**: Intel Core i7或更高（12核+推荐）
- **内存**: 16GB及以上（32GB推荐）
- **GPU**: NVIDIA RTX系列（支持CUDA）
- **硬盘**: 100GB可用空间
- **网络**: 千兆有线网卡 + Wi-Fi

#### Windows仿真机（推荐配置）
- **CPU**: Intel Core i5或更高
- **内存**: 16GB及以上
- **GPU**: 支持DirectX 12（独立显卡推荐）
- **硬盘**: 200GB可用空间
- **网络**: 千兆有线网卡 + Wi-Fi

#### 网络设备
- 千兆网线（CAT5e或更高）
- Wi-Fi路由器（可选，用于互联网访问）

### 2.2 软件依赖

#### Ubuntu工控机
```bash
操作系统: Ubuntu 22.04 LTS
ROS2版本: Humble
Python: 3.10
DDS实现: CycloneDDS或FastDDS
```

**安装ROS2 Humble**:
```bash
# 设置locale
sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

# 添加ROS2源
sudo apt install software-properties-common
sudo add-apt-repository universe
sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

# 安装ROS2 Humble
sudo apt update
sudo apt upgrade
sudo apt install ros-humble-desktop
sudo apt install ros-dev-tools

# 安装CycloneDDS
sudo apt install ros-humble-rmw-cyclonedds-cpp
```

#### Windows仿真机
```
操作系统: Windows 10/11 (64-bit)
ROS2版本: Humble (Windows版)
UE5: 5.x
MATLAB/Simulink: R2022b或更高
```

**安装ROS2 Humble (Windows)**:
1. 下载ROS2 Humble Windows安装包: https://docs.ros.org/en/humble/Installation/Windows-Install-Binary.html
2. 解压到 `C:\dev\ros2_humble\`
3. 安装Visual Studio 2022及C++开发工具

## 3. 网络配置

### 3.1 网络拓扑

```
[ Ubuntu工控机 ]              [ Windows仿真机 ]
  Wi-Fi: 172.16.21.69           Wi-Fi: 172.16.21.81
  有线:  192.168.100.1    ←网线→   有线:  192.168.100.2
       (eno2)                        (以太网)
```

### 3.2 Ubuntu工控机配置

#### 步骤1: 配置有线网卡静态IP

1. 打开设置 → Network（网络）
2. 找到Wired（有线连接）→ 点击齿轮⚙️
3. 切换到IPv4标签页
4. 设置如下:
   - **IPv4 Method**: Manual
   - **Address**: 192.168.100.1
   - **Netmask**: 255.255.255.0
   - **Gateway**: **留空**（重要！）
   - **DNS**: 留空
5. 点击Apply（应用）
6. 关闭再开启有线连接

#### 步骤2: 验证配置

```bash
# 查看IP配置
ip addr show eno2
# 应显示: inet 192.168.100.1/24

# 查看路由表
ip route
# 应看到: 192.168.100.0/24 dev eno2
#        default via 172.16.21.1 dev wlx... (Wi-Fi作为默认网关)
```

#### 步骤3: 配置CycloneDDS绑定有线网卡

```bash
# 创建配置目录
mkdir -p ~/.config/cyclonedds

# 创建配置文件
cat > ~/.config/cyclonedds/cyclonedds.xml << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<CycloneDDS xmlns="https://cdds.io/config" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xsi:schemaLocation="https://cdds.io/config https://raw.githubusercontent.com/eclipse-cyclonedds/cyclonedds/master/etc/cyclonedds.xsd">
  <Domain>
    <General>
      <Interfaces>
        <NetworkInterface address="192.168.100.1"/>
      </Interfaces>
    </General>
  </Domain>
</CycloneDDS>
EOF

# 设置环境变量（建议写入~/.bashrc）
echo "export CYCLONEDDS_URI=file://\$HOME/.config/cyclonedds/cyclonedds.xml" >> ~/.bashrc
echo "export RMW_IMPLEMENTATION=rmw_cyclonedds_cpp" >> ~/.bashrc
echo "export ROS_DOMAIN_ID=10" >> ~/.bashrc
source ~/.bashrc
```

### 3.3 Windows仿真机配置

#### 步骤1: 配置有线网卡静态IP

1. 打开 控制面板 → 网络和Internet → 网络和共享中心
2. 点击 更改适配器设置
3. 右键"以太网" → 属性
4. 双击"Internet协议版本4 (TCP/IPv4)"
5. 选择"使用下面的IP地址"，设置如下:
   - **IP地址**: 192.168.100.2
   - **子网掩码**: 255.255.255.0
   - **默认网关**: **留空**（重要！）
   - **DNS服务器**: 留空
6. 点击确定

#### 步骤2: 配置防火墙

**允许ICMP（ping）**:
1. 打开 Windows Defender 防火墙 → 高级设置
2. 左侧 → 入站规则
3. 找到"文件和打印机共享（回显请求 - ICMPv4-In）"
4. 右键 → 启用规则（专用和公用配置文件都启用）

**设置网络配置文件**:
1. 设置 → 网络和Internet → 以太网
2. 点击当前连接
3. 网络配置文件 → 选择"专用（Private）"

#### 步骤3: 配置CycloneDDS（可选）

```powershell
# 创建配置目录
mkdir C:\config\cyclonedds

# 创建cyclonedds.xml文件，内容如下:
```

**C:\config\cyclonedds\cyclonedds.xml**:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<CycloneDDS xmlns="https://cdds.io/config">
  <Domain>
    <General>
      <Interfaces>
        <NetworkInterface address="192.168.100.2"/>
      </Interfaces>
    </General>
  </Domain>
</CycloneDDS>
```

**设置环境变量**:
```powershell
# 设置系统环境变量（管理员权限）
setx CYCLONEDDS_URI "file://C:/config/cyclonedds/cyclonedds.xml" /M
setx ROS_DOMAIN_ID "10" /M
setx RMW_IMPLEMENTATION "rmw_cyclonedds_cpp" /M
```

## 6. 故障排查

### 6.1 网络连接问题

#### 问题: ping不通

**症状**: Ubuntu无法ping通Windows（或反之）

**排查步骤**:
```bash
# 1. 检查IP配置
ip addr show eno2  # Ubuntu
ipconfig           # Windows

# 2. 检查网线连接
# 确认网线两端连接牢固，网卡指示灯亮起

# 3. 检查防火墙（Windows）
# 确认ICMP规则已启用（见3.3节）

# 4. 检查路由表
ip route  # Ubuntu
route print  # Windows
```

**解决方案**:
- 重新配置静态IP（参考3.2和3.3节）
- 启用Windows防火墙ICMP规则
- 检查网线和网卡硬件

#### 问题: ROS2节点无法发现

**症状**: `ros2 node list`看不到对方机器的节点

**排查步骤**:
```bash
# 1. 检查ROS_DOMAIN_ID
echo $ROS_DOMAIN_ID  # 两台机器应相同（如10）

# 2. 检查DDS配置
echo $CYCLONEDDS_URI
cat $HOME/.config/cyclonedds/cyclonedds.xml

# 3. 抓包验证DDS流量
sudo tcpdump -i eno2 udp port 7400-7500

# 4. 检查防火墙是否阻止DDS端口
sudo ufw status  # Ubuntu
```

**解决方案**:
- 统一设置 `ROS_DOMAIN_ID=10`
- 确认CycloneDDS配置正确（参考3.2步骤3）
- 关闭或配置防火墙允许DDS通信

### 6.2 图像传输问题

#### 问题: 图像消息未发布

**症状**: `ros2 topic echo /camera/image_raw`无输出

**排查步骤**:
```bash
# 1. 检查话题是否存在
ros2 topic list | grep image

# 2. 检查发布者
ros2 topic info /camera/image_raw
# 应显示: Publishers: 1

# 3. 检查Simulink模型运行状态
# 确认Simulink仿真正在运行

# 4. 检查Bus维度配置
# 查看Simulink错误日志
```

**解决方案**:
- 启动Simulink仿真
- 检查Bus维度配置（参考DevLogs中的文档）
- 验证ROS2 Publisher模块配置

#### 问题: 图像传输频率低

**症状**: `ros2 topic hz /camera/image_raw` 显示频率 < 30Hz

**排查步骤**:
```bash
# 1. 检查网络延迟
ping 192.168.100.2

# 2. 检查CPU/内存占用
htop

# 3. 检查图像分辨率
ros2 topic echo /camera/image_raw --once
# 查看 height, width字段

# 4. 检查QoS策略
ros2 topic info /camera/image_raw -v
```

**解决方案**:
- 降低图像分辨率（推荐640×480）
- 使用有线网络（确认DDS配置）
- 优化Simulink仿真步长
- 使用BEST_EFFORT QoS（牺牲可靠性换取速度）

### 6.3 仿真问题

#### 问题: Simulink终止时卡住

**症状**: 点击停止按钮后Simulink无响应

**解决方案**:
```matlab
% 1. 强制停止仿真
set_param(bdroot, 'SimulationCommand', 'stop')

% 2. 如果仍无响应，关闭MATLAB重启

% 3. 预防措施: 在StopFcn中添加清理代码
% Model Properties -> Callbacks -> StopFcn:
% close_udp_connection();
```

**参考文档**: `DevLogs/BugFixLogs/SimulinkDynamics/Simulink_Termination_Hanging.md`

#### 问题: UE5帧率低

**症状**: UE5运行卡顿，FPS < 30

**解决方案**:
- 降低UE5渲染质量（设置 → Scalability → Low）
- 关闭不必要的后处理效果
- 减少场景中的Actor数量
- 检查GPU占用率（使用任务管理器）

## 7. 常见问题FAQ

### Q1: 如何更改ROS2通信域ID？

**A**: 修改环境变量 `ROS_DOMAIN_ID`

```bash
# Ubuntu
export ROS_DOMAIN_ID=20  # 临时设置
echo "export ROS_DOMAIN_ID=20" >> ~/.bashrc  # 永久设置

# Windows（管理员权限）
setx ROS_DOMAIN_ID "20" /M
```

**注意**: 两台机器的 `ROS_DOMAIN_ID` 必须相同才能通信。

### Q4: 支持哪些图像分辨率？

**A**: 推荐分辨率如下

| 分辨率 | 频率 | 适用场景 | 性能 |
|--------|------|---------|------|
| 100×100 | 30Hz | 测试、演示 | ⭐⭐⭐⭐⭐ 最佳 |
| 640×480 | 30Hz | 实际应用 | ⭐⭐⭐⭐ 推荐 |
| 1280×720 | 20-25Hz | 高清需求 | ⭐⭐⭐ 可用 |
| 1920×1080 | <20Hz | 不推荐 | ⭐⭐ 性能不足 |

### Q5: 如何添加新的ROS2节点？

**A**: 创建新节点的基本步骤

```bash
# 1. 在workspace中创建Python节点
cd ~/yjy_usv_ros2_cxz/control_planner
touch my_new_node.py
chmod +x my_new_node.py

# 2. 编写节点代码
```

**my_new_node.py示例**:
```python
#!/usr/bin/env python3
import rclpy
from rclpy.node import Node
from std_msgs.msg import String

class MyNewNode(Node):
    def __init__(self):
        super().__init__('my_new_node')
        self.publisher_ = self.create_publisher(String, 'my_topic', 10)
        self.timer = self.create_timer(1.0, self.timer_callback)

    def timer_callback(self):
        msg = String()
        msg.data = 'Hello from my new node!'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: {msg.data}')

def main(args=None):
    rclpy.init(args=args)
    node = MyNewNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()
```
